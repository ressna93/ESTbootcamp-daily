## 4교시 (13:00~14:00)

### 동기 / 비동기 개념 심화

#### 동기(Synchronous)

**예시: 쿠팡에서 사과 주문**

- 사과가 도착할 때까지 문 앞에서 기다리고 있음
- 사과가 도착할 때까지 다른 작업을 수행하지 않음

**예시: 세탁기**

- 세탁을 돌리기(1시간)
- 세탁이 끝날때 까지 기다린다(1시간)
- 밥 , 여가 생활 및 등

**프로그래밍 관점**

- JS 프로그램 실행의 제어권을 가지고 있는 '스레드(Thread)'가 차단된(Blocking) 상태
- 스레드가 코드를 1번 라인부터 실행하다가 사과를 주문하고 기다리는 함수를 만나면 실행을 멈춤
- 따라서 다음 라인을 실행하지 못하고 대기

#### 비동기(Asynchronous)

**예시: 쿠팡에서 사과 주문**

- 사과가 도착할 때까지 다른 작업 수행 (유튜브, 넷플릭스 등)
- 쿠팡맨이 도착하면 그때 사과를 받으면 됨

**예시: 세탁기**

- 세탁을 돌린다(1시간)
- 밥,여가생활 및 등 시간을 보내고 있는다
- 1시간뒤 세탁물을 꺼낸다

### 주요 개념 정리

#### 스레드(Thread)

- 프로그램 실행의 제어권을 가지고, 실제 실행을 담당하는 역할/객체
- 프로세스 내에서 실제 코드의 실행을 담당
- **싱글 스레드**: 프로세스 내에 스레드가 1개
- **멀티 스레드**: 프로세스 내에 스레드가 2개 이상

#### 프로세스(Process)

- 프로그램을 메모리 상에 올려서 실행시키는 단위

#### 프로그램(Program)

- 디스크 상 어딘가에 위치한 소스코드 묶음

**예시: Chrome 브라우저**

1. PC 구매 시 Chrome이 설치되어 있지 않음
2. Chrome 프로그램을 C드라이브 등의 디스크에 설치
3. 사용자가 Chrome을 더블클릭하여 실행
4. OS가 디스크에 있는 Chrome 프로그램을 찾아서 프로세스 형태로 실행
5. 프로세스는 메모리에 할당되어 동작
6. Chrome 종료 시 프로세스가 종료됨

### JavaScript와 비동기 처리

**JS의 특징**

- JS는 기본적으로 스레드가 하나로 동작 (싱글 스레드)
- 동기적으로 동작할 수밖에 없는 한계를 가지고 있음

**비동기 처리가 가능한 이유**

- JS 런타임 자체는 싱글 스레드
- JS는 브라우저에서 동작
- 브라우저는 하나의 소프트웨어로, 별도의 백그라운드 스레드들을 보유
- JS 런타임은 싱글 스레드이지만, 브라우저가 가진 별도의 스레드(Web API)들이 같이 동작하기 때문에 비동기 처리가 가능
