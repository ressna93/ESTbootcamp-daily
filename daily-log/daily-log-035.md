### 2교시 (10:00~11:00) - Node.js의 이해

#### 프로그램 vs 프로세스
- **프로그램**: 디스크 상에 설치된 코드의 묶음 (비실행 상태)
- **프로세스**: 실행 중인 프로그램 (실행 상태)

#### 서버 실행 과정
- 서버(프로그램)를 개발 → 서버 시작 → 서버 프로세스 생성
- 브라우저도 실행하면 프로세스가 됨

#### Node.js가 필요한 이유
- JavaScript를 서버에서 실행하려면 별도의 서버 프로세스가 필요
- 브라우저 밖에서 JavaScript를 실행할 수 있는 환경이 필요
- **JavaScript 런타임**: JavaScript 코드를 실행할 수 있는 실행 환경이 구성된 프로그램

#### Node.js란?
- 브라우저 내부의 JavaScript 런타임을 브라우저 밖으로 꺼내서 독립적으로 사용할 수 있게 만든 라이브러리
- JavaScript를 실행할 수 있는 JS 런타임
- 주로 서버 개발에 활용됨

#### Node.js 설치 및 기본 터미널 명령어
```bash
# 디렉토리 이동
cd [경로]

# 현재 경로의 파일/폴더 목록 확인
ls

# 현재 경로 확인
pwd

# Node.js 실행
node [실행하려는js파일]
# 예: node server.js
```

### 6교시 (15:00~16:00) - HTTP 요청 메서드

#### HTTP 요청 방식
크게 9~10가지 방식이 있지만, 본질적으로 2가지로 분류:

#### 1. GET 요청
- **조회 요청** (읽기 작업)
- 서버가 가진 데이터(html, png, mp4, json 등)를 조회, 확인하는 요청
- 예: `naver.com/index.html` 요청 → index.html 파일을 읽고 싶은 것

#### 2. POST 요청
- **데이터를 추가/수정/삭제**할 때 사용 (쓰기 작업)
- 예시:
  - 게시물 작성: 서버의 데이터베이스에 새로운 게시글 데이터 추가
  - 로그인: ID/PW를 서버에 전송

### 7교시 (16:00~17:00) - 클라이언트에서 서버로 데이터 전송

### 8교시 (17:00~18:00) - 서버 응답 처리
- 서버로부터 응답을 받았을 경우 실행시킬 이벤트 로직(`onload`)
- 응답 메시지의 body 부분에 `responseData`를 넣어서 클라이언트로 전달
- 클라이언트는 서버로부터 응답받은 데이터를 화면에 렌더링

##  핵심 개념 정리

### Node.js
- 브라우저 밖에서 JavaScript를 실행할 수 있게 해주는 JavaScript 런타임
- 서버 개발에 주로 활용

### Express.js
- Node.js 기반 웹 서버 개발 프레임워크
- 서버 개발을 보다 추상화되고 간편하게 할 수 있도록 도와줌

### HTTP 요청의 2가지 핵심
1. **GET**: 데이터 조회 (읽기)
2. **POST**: 데이터 추가/수정/삭제 (쓰기)

### 클라이언트-서버 통신
- Request(요청) 객체와 Response(응답) 객체를 통한 데이터 교환
- 요청 메시지와 응답 메시지의 body 부분에 데이터 포함
