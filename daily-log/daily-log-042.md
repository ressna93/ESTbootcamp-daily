## 2교시 (10:00~11:00)

### 크로스 브라우징과 jQuery

#### 1. 크로스 브라우징이란?

- **개념**: 다양한 웹 브라우저에서 동일하게 작동하고 표시되도록 하는 기술
- 단순히 똑같이 보이게 하는 것이 아니라, 각 브라우저에서 사용자가 의도한 기능을 정상적으로 사용할 수 있도록 하는 것

#### 2. jQuery란?

- JavaScript를 쉽게 사용하기 위한 라이브러리

#### 3. 모던 개발환경에서 jQuery의 필요성

- 기존 프로젝트 유지보수
- 이미 완성된 페이지 관리

#### Q. jQuery는 무겁다?

- **JS 번들 사이즈 증가**: 파일 크기 부담
- **브라우저 연산 처리 부담**: jQuery 코드 동작을 위한 추가 연산

## 4교시 (13:00~14:00)

### 브라우저 렌더링

#### 1. 렌더링 개요

- **페이지 렌더링에 필요한 필수 리소스**
- **점진적 렌더링(Progressive Rendering)**

#### 2. CRP (Critical Rendering Path)

- **개념**: 브라우저가 HTML, CSS, JavaScript를 화면에 픽셀로 변환하는 과정
- **CRP 수행 단계**: DOM 생성 → CSSOM 생성 → Render Tree 생성 → Layout → Paint
- **특징**: 렌더링은 지속적으로 반복 수행되는 과정

## 5교시 (14:00~15:00)

### 렌더링 차단 리소스

#### CRP 모듈 설치

- 실습 환경 구성

#### Parser-blocking 실습

렌더링 작업을 차단하는 리소스들

#### 1. 렌더링 차단 리소스 (Render-blocking Resources)

- 페이지의 초기 렌더링을 차단하는 리소스

#### 2. 파서 차단 리소스 (Parser-blocking Resources)

- HTML 파싱을 중단시키는 리소스

#### 5. Render Blocking이 필요한 이유

**Q. CSS 리소스가 렌더링을 차단하는 리소스로 동작하는 이유는 무엇일까?**

**A.** 렌더링을 차단하는 리소스인 CSS는 CSSOM이 구성될 때까지 브라우저의 렌더링을 차단시키는데, 이를 통해 화면 깜빡임 현상(FOUC)을 방지할 수 있음

#### FOUC (Flash of Unstyled Content)

- **현상**: CSS가 네트워크를 통해 다운로드 완료 후 스타일이 뒤늦게 적용되면서, 스타일이 적용되지 않았던 페이지가 스타일이 적용된 페이지로 갑자기 교체되는 과정에서 페이지가 일시적으로 깜빡거리는 현상
- **문제점**: 사용자의 혼란 야기
