### 2️ 터미널(Command Line) 기초

**7교시 (16:00~17:00)**

개발자에게 필수적인 터미널 사용법을 배웠습니다.

**터미널이란?**

- 텍스트 명령어로 컴퓨터를 제어하는 인터페이스
- CLI (Command Line Interface)라고도 불림
- GUI보다 빠르고 강력한 작업 수행 가능

**기본 명령어**

```bash
# 현재 위치 확인
pwd                    # Print Working Directory

# 디렉토리 이동
cd <폴더명>            # Change Directory
cd ..                  # 상위 폴더로 이동
cd ~                   # 홈 디렉토리로 이동

# 파일/폴더 목록 보기
ls                     # Linux/Mac
dir                    # Windows

# 폴더 생성
mkdir <폴더명>

# 파일 생성
touch <파일명>         # Linux/Mac
echo. > <파일명>       # Windows

# 파일/폴더 삭제
rm <파일명>            # Linux/Mac
del <파일명>           # Windows
```

**개발자가 터미널을 쓰는 이유**

- Git으로 버전 관리
- npm으로 패키지 설치 및 관리
- 프로젝트 빌드 및 실행
- 서버 배포 및 관리
- 자동화 스크립트 실행

### 3️⃣ 프로그래머스 코딩테스트 레벨 0

**7~8교시 (16:00~18:00)**

프로그래머스 레벨 0 문제들을 풀며 알고리즘 사고력을 키웠습니다.

**레벨 0의 특징**

- 기초 문법을 활용한 간단한 문제들
- 문자열, 배열, 수학 연산 등 기본 자료구조 활용
- 문제 해결 능력의 기초를 다지는 단계

**문제 풀이 접근법**

1. **문제 이해하기**

   - 입력값과 출력값 파악
   - 제약 조건 확인
   - 예제를 통해 패턴 파악

2. **해결 방법 구상하기**

   - 어떤 자료구조를 쓸지 결정
   - 어떤 메서드나 알고리즘이 필요한지 생각
   - 의사코드(pseudocode)로 로직 정리

3. **코드 작성하기**

   - 단계별로 차근차근 구현
   - 변수명을 명확하게 작성
   - 간결하고 읽기 쉬운 코드 지향

4. **테스트 및 디버깅**
   - 주어진 테스트 케이스 확인
   - 엣지 케이스(극단적인 경우) 고려
   - 오류 발생 시 console.log로 디버깅

**자주 사용한 메서드/패턴**

```javascript
// 문자열 분리
str.split("");

// 배열 순회 및 변환
arr.map((item) => item * 2);

// 배열 필터링
arr.filter((item) => item > 0);

// 배열 정렬
arr.sort((a, b) => a - b);

// 배열 합계
arr.reduce((sum, cur) => sum + cur, 0);
```

**코딩테스트 팁**

- 처음부터 완벽한 코드를 작성하려 하지 말기
- 일단 작동하는 코드를 만든 후 리팩토링
- 시간복잡도를 항상 염두에 두기
- 다양한 풀이 방법을 고민해보기
