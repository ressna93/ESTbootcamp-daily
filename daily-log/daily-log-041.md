## HTTPS

### 개요

- HTTP + TLS
- HTTP만으로 발생할 수 있는 보안 문제를 해결

### HTTP에서 발생할 수 있는 문제

1. **데이터 도청 문제**
2. **데이터 변조 문제**
3. **상대방 신뢰 문제**

### HTTPS가 해결하는 방식

1. **암호화 (Encryption)**
2. **해시화 (Hashing)**
3. **인증 및 인증서 발급 (Certificate)**

## SSL/TLS

### SSL의 발전 흐름

- SSL: 웹에서 데이터를 안전하게 전달하기 위한 출발점
- TLS: 구조적 문제점들을 보완하고 현대화, 표준화시킨 프로토콜

### TLS의 개선 사항

- 구조적 취약점 제거 및 보완
- 암호 기법 강화
- 성능 개선
- 다양한 공격 패턴 방어

### TLS 세부 단계

1. 인증서 준비
2. 핸드셰이크 (Handshake)
3. 데이터 암호화 통신

---

## 암호화 알고리즘

### 1. 대칭키 암호화 알고리즘

- 키가 1개
- 빠른 처리 속도

### 2. 공개키 암호화 알고리즘

- 키가 2개 (공개키, 개인키)
- 대칭키 암호화의 단점을 보완
- 처리 과정에서 시간이 더 소요됨

### TLS에서의 활용

- **TLS 핸드셰이킹 과정에서 대칭키와 공개키를 함께 사용**
- 공개키로 안전하게 대칭키를 교환
- 실제 데이터 통신은 대칭키로 수행

## CORS (Cross-Origin Resource Sharing)

### CORS란?

- 교차 출처 리소스 공유
- 브라우저의 보안 정책과 관련된 개념

### CORS 오류 발생 조건

- `app.js`에서 `fetch`를 사용해 다른 출처의 서버로 요청 전송 시
- Node.js 서버나 NCP 서버로 요청 시 브라우저 콘솔에 CORS 관련 에러 발생

### CORS vs SOP

- **SOP (Same-Origin Policy)**: 동일 출처 정책
- **CORS**: SOP의 제한을 안전하게 우회하는 메커니즘

- CORS가 등장한 계기

- 해결 방법 (여러 가지 방법 존재)
- CORS 오류 실습 및 해결
